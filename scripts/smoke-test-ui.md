# V2.3 冒烟测试 — UI 层手动清单

> 跟 `smoke-test.js`(云函数端)配套,这一份是**小程序 UI 层**手动测的清单。云函数测不到的(tabBar 生命周期、客户端 TTS 重试、状态机残留弹窗)都在这里。

## 前置

- [ ] 已部署 V2.3 代码(`cloudfunctions/main` 已上传)
- [ ] 已配置云函数环境变量(4 项,新密钥)
- [ ] 已登录(已走完 `wxLogin`)
- [ ] 微信开发者工具 → 模拟器 → 打开 Console(看日志用)

## 冒烟测试用例

### TC-1:首页"已掌握"数字一致性

**目的**:验证 `getStats` 改用 `learning_progress` 查后,首页和列表页数字一致(CLAUDE.md L177 老 bug 根除)

**步骤**:
1. 首次进入首页,记下"已掌握"数字 N
2. 点击首页统计卡片的"已掌握"区域,跳到列表页
3. 看列表页顶部的"(N个)"
4. 两个数字应该**完全一致**

**通过条件**:两个数字相等

---

### TC-2:从已掌握点字进入学习页(修 C5)

**目的**:验证 tabBar 页面 onShow 消费 fromMasteredChar(昨天修的 bug)

**步骤**:
1. 切到首页,点"已掌握"进入列表页
2. 选**任意一个**字(不要是"一")进入学习页
3. 看 Step1 显示的字

**通过条件**:显示的字 = 选中的字(不是"一")

---

### TC-3:已掌握 → 学完一个字 → 点其他字(修 C6)

**目的**:验证状态机残留不会导致"学会了"弹窗

**步骤**:
1. 列表页选一个"未掌握"或"小复习"字(若没有,先学一个)
2. 走完四步,看到"学会了"弹窗(完成态)
3. **不点"继续"**,直接点 tabBar 切到别的页(index/profile),再回 mastered
4. 选**另一个**字进入学习页
5. 应该从 **Step1 释义**开始,不应该直接看到"学会了"弹窗

**通过条件**:新字正常走 Step1

---

### TC-4:新字模式触发配额 → 切到已掌握(修 C7)

**目的**:验证 dailyQuotaReached 不残留

**步骤**:
1. **如果**之前测试导致学完 5 个新字(触发"今日新字已达标"),进入学习 tab
2. 应该看到"今日新字已达标"卡片(不去复习,返回首页)
3. 进已掌握,点任意字
4. 进入学习页应该**看不到**"今日新字已达标"卡片

**通过条件**:配额卡片消失

**注意**:如果今日新字没到 5,这个用例需要在"5 个新字"状态下测;或者临时在 Console 跑 `s.callCloud('recordLearn', ...)` 5 次,触发配额

---

### TC-5:清除学习数据按钮(V2.3 新功能)

**目的**:验证 settings 清除按钮真的能清,且清完能重新登录

**步骤**:
1. 进设置 tab(个人中心 → 右上角设置图标)
2. 滚到底,点"清除学习数据"(橙色)
3. 弹出"⚠️ 清除学习数据"确认框
4. 点"确定清除"
5. 等待 "已清除" toast + 自动跳回首页
6. 首页应该显示"已掌握 0、星星 0、小红花 0",需要重新登录

**通过条件**:
- toast 出现 "已清除"
- 跳回首页
- 重新登录后数字全是 0

---

### TC-6:TTS 音频播放(P0-4)

**目的**:验证百度 TTS 能正常播放

**步骤**:
1. 进学习 tab,任意一个字
2. 点"🔊 听发音"
3. 应该能听到发音
4. 重复几次(测试稳定性)

**通过条件**:听到清晰发音

**降级测试**(可选):
- 临时在云开发控制台把 `BAIDU_API_KEY` 改错,看是否显示"播放失败" toast,再改回正确值

---

### TC-7:复习页 V2.2 复习链路

**目的**:验证 recordLearn 同步 learning_progress 后,新字能进复习页(核心机制)

**步骤**:
1. 学一个新字(走完四步)
2. **等待 next_review_date** —— 默认是今天+1天,所以**实际上你可以手动调 next_review_date**
3. 或者:用云端调试,直接调 `recordReview({...isCorrect: true})` 把那个字的 next_review_date 改成今天
4. 进复习 tab,看复习队列有没有这个字

**通过条件**:复习队列出现那个字

**简化方法**:
- 跳过这个用例,等"明天"自然验证(学一个字,等 1 天看复习列表)
- 或直接信任 smoke-test.js 里 4.3 的断言(已经验证 next_review_date 字段被正确设置)

---

### TC-8:复习页 Box 升降反馈(V2.2)

**目的**:验证复习答题后 Box 等级 + 状态变迁有提示

**步骤**:
1. 进复习 tab,任意一个字
2. 选对一个(应该听到连击提示 + 星星粒子)
3. 选错一个(应该看到渐进式提示 + 振动)
4. 答对 5 连击,看是否触发"超级连击🔥🔥🔥"动画

**通过条件**:反馈动画 + 提示都正常出现

---

### TC-9:首页成长等级 + 每日进度(R-14)

**目的**:验证首页顶部成长等级、剩余字数、待复习数都正常显示

**步骤**:
1. 进首页
2. 看顶部:成长等级(🌱/🌿 等)+ 距下一级 N 字
3. 看中部"已掌握、星星、小红花"统计(数字应和列表页一致,见 TC-1)
4. 看"今日进度"区:新字 X/Y
5. 如果有"今日待复习"卡片,点进去应该跳到复习 tab

**通过条件**:所有数字与实际学习状态一致

## 已知问题/限制

- TC-7 的"等 1 天"是真实场景下的终极验证,但手动测试不方便;**简化方案是看 smoke-test.js 4.3 通过即认为 OK**
- TTS 测试受网络影响,如果有"播放失败"toast,**先检查网络**,再考虑是不是百度 token 失效(会自动重试 1 次)
- 清除数据按钮是"硬删除",**清完不可恢复**。如果想保留测试数据,**先备份**(用 smoke-test.js 在清之前跑一遍场景 1-6 记下数据)
