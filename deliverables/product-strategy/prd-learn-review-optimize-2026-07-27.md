## 📌 TL;DR（执行摘要）

当前识字小程序（shizi）学习与复习模块存在致命数据缺陷与认知设计缺陷：`Math.random()` 导致56%假阳性率、复习数据闭环断裂、学习流程违背3-6岁儿童"再认先于回忆"的认知规律。本 PRD 提出以**四步递进教学 + Leitner Box 间隔重复 + 渐进错误提示**为核心的优化方案，分三阶段交付（P0紧急修复→P1机制建设→P2体验提升），预计可将知识30天保持率从约10%提升至70%+，同时消除数据信任危机并建立科学的掌握判定体系。

## 1. 产品目标

### 目标1：修复数据信任，消除假阳性
> 将当前56%的综合假阳性率降至5%以下，确保"掌握"判定真实可靠，让家长和产品团队重新信任学习数据。

### 目标2：建立科学的间隔重复体系
> 用Leitner Box替代随机复习，将30天知识保持率从约10%提升至70%+，同时打通复习数据闭环，让每次复习结果都反馈到学习进度。

### 目标3：对齐儿童认知规律的递进教学
> 重构学习流程为"再认→回忆→动觉→产出"四步递进，匹配3-6岁儿童工作记忆容量（3-5项）和注意力时长（5-8分钟），让学习过程本身符合认知科学而非违背它。

**目标正交性说明**：目标1解决"数据准不准"、目标2解决"忘不忘"、目标3解决"学得对不对"，三者互不依赖但协同增效。

## 2. 用户故事

### US-1：家长的"虚假安心"困境
> 作为家长，我发现孩子被标记为"已掌握"50个字，但实际测试时只认得20个，我无法信任小程序的学习报告，也不知道哪些字需要重点巩固。

**关联目标**：目标1（消除假阳性）
**验收**：修复后"掌握"状态的字，线下抽检正确率≥90%

### US-2：孩子的"学完就忘"困境
> 作为5岁的孩子，我昨天学了5个新字，今天再看到完全不认识了，妈妈让我重新学，我觉得好无聊，不想学了。

**关联目标**：目标2（间隔重复）
**验收**：新学汉字在1天/3天/7天后的识别正确率分别≥85%/75%/65%

### US-3：孩子的"直接跟读"挫败感
> 作为4岁的孩子，刚看到一个新字就要马上跟读出来，我说错了小程序直接显示正确答案，我不知道为什么是这个音，只能死记硬背。

**关联目标**：目标3（递进教学）
**验收**：新字学习流程完成率≥90%（当前约70%）；学习过程退出率下降30%

### US-4：孩子的"反复出错"无助感
> 作为4岁的孩子，"己"和"已"我总是搞混，每次复习都答错，小程序只是告诉我答错了，从不帮我搞清楚区别，我觉得自己好笨。

**关联目标**：目标3（渐进错误提示+错因诊断）
**验收**：错字二次正确率≥60%；错误后继续尝试率≥80%

### US-5：家长的"学习节奏"焦虑
> 作为家长，我不知道孩子一天该学多少个字，小程序没有限制，孩子有时候一口气学10个，第二天全忘了，我担心学习效率太低。

**关联目标**：目标2+3（每日学习量控制）
**验收**：系统限制新字3-5个/次、5-8个/天，单次学习5-8分钟

## 3. 用户研究洞察（来自瑞思）

### 3.1 核心认知规律

| 维度 | 3-4岁 | 5-6岁 | 设计含义 |
|------|-------|-------|----------|
| 工作记忆容量 | 3-4个项目 | 4-5个项目 | 每次新学≤3-5个字 |
| 注意力时长 | 3-5分钟 | 10-15分钟 | 单次学习5-8分钟 |
| 识字认知阶段 | 再认为主 | 再认→回忆过渡 | 先"看字选音/义"再"跟读录音" |
| 正字法意识 | 隐式感知 | 开始发展 | 不显性教偏旁，大量视觉暴露 |

### 3.2 间隔重复的儿童特殊需求
- 儿童遗忘速度显著快于成人，**初始间隔必须缩短**
- 推荐间隔序列：1小时→1天→3天→7天→14天→30天
- 首次复习（1小时后）对儿童尤其关键——当前系统完全缺失

### 3.3 动机维持的关键发现
- **关联感（Relatedness）**是自我决定理论中效应量最大的维度（g=1.776）
- 口头表扬效果 > 物质奖励；过程表扬（"你很努力！"）> 结果表扬（"你真棒！"）
- 应培养**成长型思维**：将失败归因于"还需要练习"而非"你不聪明"
- 避免公开排行榜，使用**个人最佳记录**

### 3.4 错误处理的认知原则
- 直接给答案 = 丧失学习机会
- 渐进提示让儿童在辅助下自主发现答案，记忆深度显著更深
- 答错3次以上应自动降级到再认模式（降低认知负荷）

### 4.2 关键对标结论

1. **递进教学是行业标配**：头部产品4-7步，我们仅2步，严重落后
2. **间隔重复是留存关键**：古古识字3月留存92%（行业平均65%），唯一采用间隔重复的竞品
3. **错误处理是差异化机会**：仅古古识字有渐进提示，8款竞品中7款直接给答案
4. **游戏化存在拐点**：洪恩游戏化最强但200字后学习效率下降，游戏应服务于学习而非替代
5. **微信生态是我们独有优势**：推送提醒复习、社交分享成就、家长端同步——独立App竞品均无法实现

### 4.3 差异化定位

**"3步学会，5次不忘"**
- 3步学会：递进教学3步核心步骤（再认→动觉→产出），让孩子真正理解而非死记
- 5次不忘：间隔重复5次复习（1天→3天→7天→14天→30天），科学对抗遗忘

## 5. 数据依据（来自数析）

### 5.1 当前系统致命缺陷

| 编号 | 问题 | 位置 | 影响 | 严重级别 |
|------|------|------|------|----------|
| D-1 | 看字说音Math.random()判定 | review.js L386 | 70%概率被标记正确，100%数据无效 | P0 |
| D-2 | 学习ASR失败随机fallback | learn.js L275/280/287, 云函数L668-683 | 约30%学习过程受影响 | P0 |
| D-3 | 间隔重复完全缺失 | 全局 | 30天知识保持率仅约10% | P1 |
| D-4 | review_logs只写不读 | review模块 | 复习数据不反馈到学习进度 | P1 |
| D-5 | recordReview不更新进度 | 云函数 | 复习与学习进度断裂 | P1 |
| D-6 | "3次正确=掌握"模型过简 | 学习判定逻辑 | 不区分记忆深度，无跨天验证，无退步 | P2 |
| D-7 | 复习优先级算法缺陷 | 复习排序 | todayLearned权重过高，缺时间衰减 | P2 |
| D-8 | 全量加载性能隐患 | 数据加载 | 字量增长后性能瓶颈 | P2 |

### 5.2 假阳性率分析

```
综合假阳性率 ≈ 56%

计算依据：
- 看字说音模式：Math.random() > 0.3 → 标记正确 → 假阳性率70%
- ASR失败fallback：随机给分 → 假阳性率50%
- 受影响比例：看字说音约40% + ASR失败约30% → 综合约56%
```

**这意味着超过一半的"掌握"可能是假的，所有基于此数据的分析、报告和决策都不可信。**

### 5.3 当前数据模型缺陷

```
当前 learning_progress 字段：
  openid, char_id, status, correct_count, wrong_count, last_review_date
  ❌ 缺少: first_learn_date, last_correct_date, next_review_date,
          review_interval, box_level, 掌握验证维度
```

```
│  │ 学习3-5字 │                                  │ 小复习   │  │
│  │ 后触发    │─────────────────────────────────▶│(本组字)  │  │
│  └──────────┘                                  └──────────┘  │
│                                                      │      │
│                                                      ▼      │
│                                               ┌──────────┐  │
│                                               │ 下一组字  │  │
│                                               │ 或结束    │  │
│                                               └──────────┘  │
└─────────────────────────────────────────────────────────────┘
```

**Step 1 - 展示释义**（再认准备阶段）
- 大字展示 + 生动配图/动画
- 标准读音播放
- 1-2个组词示例
- 约30秒，无需操作，纯输入

**Step 2 - 再认练习**（再认验证阶段）
- 题型：看字选音（3选1）/ 看字选义（3选1）/ 看字选图（3选1）
- 答对：进入Step 3
- 答错：显示正确答案并强调关联（如"日的图片是太阳☀️"），再给一次机会
- 约30秒

**Step 3 - 描红书写**（动觉编码阶段）
- 手指跟随笔顺引导描红
- 完成描红后显示完整字+再次播放读音
- 不要求书写完美，重在动觉记忆
- 约40秒

**Step 4 - 跟读产出**（回忆验证阶段）
- 显示字，提示跟读
- ASR识别判断发音
- ASR失败：引导重试（最多3次），仍失败则降级为"听音选字"选择题
- 约30秒

**小复习规则**：
- 每学完3-5个新字后触发
- 仅测试本组字的再认能力（Step 2级别题型）
- 答错的字标记为"需加强"，在今日复习队列中优先出现

### 7.2 复习间隔重复流程

```
┌──────────────────────────────────────────────────────────────────┐
│                    Leitner Box 间隔重复流程                        │
├──────────────────────────────────────────────────────────────────┤
│                                                                  │
│  ┌────────┐  答对   ┌────────┐  答对   ┌────────┐  答对         │
│  │ Box 1  │───────▶│ Box 2  │───────▶│ Box 3  │───────┐       │
│  │ 1天    │        │ 3天    │        │ 7天    │        │       │
│  └────────┘        └────────┘        └────────┘        │       │
│      ▲                                                ▼       │
│      │ 答错     ┌────────┐  答对   ┌────────┐  答对   ┌──────┐ │
│      └──────────│ Box 5  │◀───────│ Box 4  │◀───────│Box 3 │ │
│                 │ 30天   │        │ 14天   │        │      │ │
│                 └────────┘        └────────┘        └──────┘ │
│                      │                               ▲        │
│                      │ 任意Box答错                    │        │
│                      └──────── 降回 Box 1 ───────────┘        │
│                                                                  │
│  复习优先级排序：                                                  │
│  ┌────────────────────────────────────────────────────┐         │
│  │ priority = urgency_score * 0.5                     │         │
│  │          + difficulty_score * 0.3                  │         │
│  │          + random_score * 0.2                      │         │
│  │                                                    │         │
│  │ urgency_score = max(0, (today-next_review_date)    │         │
│  │                     / max_interval) * 100          │         │
│  │ difficulty_score = (1 - historical_accuracy) * 100 │         │
│  │ random_score = Math.random() * 20                  │         │
│  └────────────────────────────────────────────────────┘         │
│                                                                  │
│  "3+2"每日学习模式：                                               │
│  ┌─────────────┐  ┌─────────────┐  ┌──────────────┐            │
│  │ 3个新字      │  │ 2个复习字    │  │ 动态调整      │            │
│  │ (四步递进)   │  │ (间隔重复)   │  │ 复习量大时     │            │
│  │             │  │             │  │ 减少新字数量   │            │
│  └─────────────┘  └─────────────┘  └──────────────┘            │
└──────────────────────────────────────────────────────────────────┘
```

**复习流程步骤**：
1. 用户进入复习模块
2. 系统计算所有待复习字的优先级分数（按上述公式）
3. 按优先级从高到低排序，取前N个（N=每日复习量上限）
4. 每个字随机选择一种复习题型
5. 用户答题
6. 根据答题结果更新box_level和next_review_date
7. 答题过程应用渐进式错误提示

**复习量自适应**：
- 当待复习字 > 10个时：减少当日新字数量（最低0个新字）
- 当待复习字 > 20个时：仅做复习，暂停新字学习
- 每日复习上限15个，防止过载

### 7.4 掌握状态变迁图

```
┌────────────────────────────────────────────────────────────────┐
│                    五级掌握状态变迁                               │
├────────────────────────────────────────────────────────────────┤
│                                                                │
│  ┌─────┐    首次学习     ┌───────┐    再认2次     ┌─────────┐ │
│  │ new │──────────────▶│seeing │────────────▶│familiar │ │
│  │未接触│               │初识   │              │熟悉      │ │
│  └─────┘               └───────┘              └─────────┘ │
│                             │                       │       │
│                             │ 答错(任何阶段)         │       │
│                             ▼                       │       │
│                        ┌─────────┐                  │       │
│                        │ 退回     │◀─────────────────┤       │
│                        │ 前一级   │                  │       │
│                        └─────────┘                  │       │
│                                                      │       │
│                              回忆2次+间隔1次          │       │
│  ┌─────────┐    ◀───────────────────────────────────┘       │
│  │mastered │    回忆2次(跟读/描红)+跨天间隔验证1次             │
│  │掌握     │                                                  │
│  └─────────┘                                                  │
│       │                                                        │
│       │ Box5+连续3次间隔复习正确                                │
│       ▼                                                        │
│  ┌──────┐                                                      │
│  │solid │                                                      │
│  │牢固  │                                                      │
│  └──────┘                                                      │
│                                                                │
│  掌握条件详解：                                                  │
│  ┌──────────────────────────────────────────────────┐          │
│  │ new → seeing:     完成四步递进学习                  │          │
│  │ seeing → familiar: 再认正确2次(不同题型)             │          │
│  │ familiar → mastered: 回忆正确2次(跟读/描红)          │          │
│  │                    + 跨天间隔验证1次正确              │          │
│  │ mastered → solid:  Box5 + 连续3次间隔复习正确        │          │
│  │                                                    │          │
│  │ 降级规则:                                           │          │
│  │ · 任意阶段连续2次错误 → 降一级                        │          │
│  │ · mastered/solid 连续2次间隔复习错误 → 降为familiar   │          │
│  │ · 30天未复习 → solid降为mastered                     │          │
│  └──────────────────────────────────────────────────┘          │
└────────────────────────────────────────────────────────────────┘
```

**关键设计决策**：
- **5次多维度验证**替代原来的"3次正确=掌握"：再认2次 + 回忆2次 + 间隔1次 = 5次
- **跨天验证**：familiar→mastered 必须有至少1次跨天正确，排除短期记忆假象
- **退步机制**：连续2次错误即降级，不会"只升不降"
- **"部分掌握"**：seeing和familiar作为中间状态，避免"未掌握→已掌握"的二元跳跃

## 8. Non-goals（明确不做什么）

| 编号 | 不做什么 | 原因 |
|------|---------|------|
| NG-1 | 不做重度游戏化（如洪恩的跑酷/对战） | 认知研究显示游戏化200字后学习效率下降；开发成本高；与"学字"核心定位冲突 |
| NG-2 | 不做显性偏旁部首教学 | 3-6岁正字法意识由阅读驱动，显性教学效果差；通过高亮偏旁隐式感知即可 |
| NG-3 | 不做AI自适应学习路径（如动态调整教学步骤数） | 当前数据基础不足以训练模型；四步递进对3-6岁已是最佳实践；过度工程化 |
| NG-4 | 不做多用户/多孩子切换 | 超出本次优化范围；微信生态下一openid一孩子是合理假设 |
| NG-5 | 不做社交学习功能（好友PK/学习小组） | 认知研究显示公开比较损害内在动机；3-6岁无需社交竞争 |
| NG-6 | 不做自定义学习计划（家长自选字表） | 与间隔重复算法冲突（算法应决定复习节奏）；增加状态管理复杂度 |
| NG-7 | 不做本次重构以外的模块优化（如首页/家长中心/报告） | 控制范围，聚焦学习+复习核心链路 |
| NG-8 | 不做离线模式 | 微信云开发架构依赖网络；离线缓存增加状态同步复杂度；3-6岁使用场景有网络 |

**停车场（好想法但不在当前范围）**：
- 🅿️ 笔顺动画教学（可作为Step 3增强，P3考虑）
- 🅿️ 语音情感识别（判断孩子是否沮丧，动态调整难度）
- 🅿️ 家长端实时学习看板（当前微信生态整合仅做推送）
- 🅿️ 自定义字表（需先验证间隔重复效果）
